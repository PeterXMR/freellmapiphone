# Building a signed release APK (Phase 6)

This produces an installable, sideloadable Android APK signed with a dedicated
**app-signing keystore** — separate from the Android Keystore that `expo-secure-store`
uses to hold API keys on-device. The two never mix: one signs the package, the
other encrypts user secrets.

## TL;DR

```bash
cd mobile
npm run build:apk            # signed release APK -> android/app/build/outputs/apk/release/
npm run build:apk -- --clean # force a clean `expo prebuild` first
```

Then install on a device with USB debugging enabled:

```bash
adb install -r android/app/build/outputs/apk/release/app-release.apk
```

## Prerequisites

| Requirement | Notes |
| --- | --- |
| **JDK 17** | RN 0.79 / AGP / Gradle 8.13 do **not** run on JDK 21+. `build-apk.sh` auto-selects JDK 17 via `/usr/libexec/java_home -v 17`; install Temurin or Oracle JDK 17 if missing. |
| **Android SDK** | Defaults to `~/Library/Android/sdk`. Override with `ANDROID_HOME`. Needs `build-tools` + `platform-tools`. |
| **Signing keystore** | `credentials/release.keystore` + `credentials/keystore.properties` (both gitignored). See below. |

## How signing is wired (and why it survives `expo prebuild`)

`android/` is **gitignored** — it is regenerated by `expo prebuild`, so editing
`android/app/build.gradle` by hand is wiped on the next prebuild. Instead, the
release `signingConfig` is injected at prebuild time by the Expo config plugin
[`plugins/withReleaseSigning.js`](../plugins/withReleaseSigning.js), registered in
`app.json`. This keeps a *signed* build reproducible from source **and** keeps the
tracking repo merge-clean (no upstream file is touched).

The plugin reads `credentials/keystore.properties`:

```properties
storeFile=credentials/release.keystore   # path relative to mobile/
storePassword=...
keyAlias=freellmapiphone-release
keyPassword=...
```

When that file is **absent** (CI, or a contributor without the keystore), the
release buildType falls back to the debug key, so `expo prebuild` and debug builds
still work — only the signed release APK needs the real keystore.

## Generating / rotating the keystore

```bash
cd mobile
keytool -genkeypair -v \
  -keystore credentials/release.keystore \
  -alias freellmapiphone-release \
  -keyalg RSA -keysize 2048 -validity 10000 \
  -storepass <store-pass> -keypass <key-pass> \
  -dname "CN=freellmapiphone, OU=mobile, O=PeterXMR, C=US"
```

> ⚠️ **Back the keystore up out-of-band.** It is the app's cryptographic identity.
> If you lose it you cannot ship an update that overwrites a previously-installed
> copy (Android refuses to replace an app with one signed by a different key), and
> you could never publish under the same Play listing. It is gitignored on purpose
> — store it in a password manager / encrypted backup, not in the repo.

## Verifying the output

```bash
$ANDROID_HOME/build-tools/*/apksigner verify --print-certs \
  android/app/build/outputs/apk/release/app-release.apk
```

`build-apk.sh` runs this automatically and prints the signer certificate.

## Distribution

v1 ships as a **sideloaded APK** (`adb install` or transfer the file and tap to
install with "install unknown apps" enabled). A Play Store `.aab` release is
deferred (YAGNI for v1) — switching to it later means building `bundleRelease`
instead of `assembleRelease` and enrolling the upload key in Play App Signing.
